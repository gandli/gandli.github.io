### ws:// vs wss:// Debate

After the upgrade, the browser reported an insecure `ws://` connection warning. I initially considered `wss://` + SSH tunnel, but after some trial and error, I returned to the original point: **`ws://` is secure for localhost**.

Data on the local loopback address does not pass through the network stack, so there is no possibility of man-in-the-middle attacks. Encryption is redundant. The final solution:

```json
{
  "gateway": {
    "port": 18789,
    "bind": "loopback"
  }
}
```

Simple, stable, and won't fail due to SSH tunnel disconnection.
