### Version Upgrade

I upgraded from `2026.2.17` to `2026.2.19-2` using `brew upgrade`, which fixed many bugs.

### ws:// vs wss:// Debate

After the upgrade, I discovered that the browser reported an insecure connection warning for `ws://`. I considered switching to `wss://` with an SSH tunnel but eventually returned to the original approach: **using `ws://` on localhost is secure**.

Local loopback addresses do not go through the network stack, so there is no risk of man-in-the-middle attacks. Encryption is unnecessary. The final solution is:

```json
{
  "gateway": {
    "port": 18789,
    "bind": "loopback"
  }
}
```

This approach is simple, stable, and won't crash due to SSH tunnel disconnections.
